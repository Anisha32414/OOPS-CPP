# Destructor :- It is opposite of constructor.

-constructor is used to allocate the memory to the objects.
-destructor is used to deallocate the memory assigned to the objects.

-Just like constructor, destructor are also inbuilt in cpp but a programmer can also make it by themselve.
    *syntax:- ~class_name(){
                   cout<<"destructor is called..memory deallocated";
              }

- Destructor is important to avoid memory leakage.
    **memory leakage:- A memory leak happens when a computer program requests memory from the operating system to 
                       perform a task, but fails to release or "return" that memory when it is no longer needed. 
                       Over time, this causes the program to consume more and more RAM, eventually leading to 
                       system slowdowns or crashes.

- The deconstructors are used to deallocate the memory allocated with help of "new" in objects.

- The deconstructor is called when the main() end.

Ex. class teacher{
    private: 
        double salary;
    public: 
        string name;
        string dept;
        int age;

        teacher(){            ----> constructor
            cout<<"new object created"<<endl;
        }

        ~teacher(){           ----> destructor
            cout<<"deallocating object memory"<<endl;
        }

        void set_info(string name,string dept,int age,double salary){
            this->name=name;
            this->dept=dept;
            this->age=age;
            this->salary=salary;
        }
        void get_info(){
            cout<<name<<endl<<dept<<endl<<age<<endl<<salary;
        }
    };
